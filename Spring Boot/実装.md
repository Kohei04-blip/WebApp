最初は以下のような書き方を意識する。  
```

4. Controller：NodeController.java

```
理由は、下から積み上げると理解しやすいから。

```
DBの形  （entity）
↓
DB操作  （repository）
↓
処理  （service）
↓
URL  （controller）
↓
画面  （view）
```

## Entity
  

### 方針

- 実装する前に何が必要かを考える

- クラス：  
Node.javaは、DBの`Node`テーブルに対応するクラス。

- フィールド：
```
id
title
content
learnedAt
createdAt
```
- アノテーション：

```java
@Entity
@Id
@GeneratedValue
```
意味は以下のようなイメージ
```
@Entity → DBテーブルになるクラス
@Id → 主キー
@GeneratedValue → idを自動採番
```


### 実装する

```java

package com.example.reviewapp.entity;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;

import java.time.LocalDate;
import java.time.LocalDateTime;

@Entity
public class Node {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    private String content;

    private LocalDate learnedAt;

    private LocalDateTime createdAt;
}

```


▶️解説  

- `package com.example.reviewapp.entity;`   
　　▶️このファイルが`entity`フォルダの中にあることを表す。  
  
- `import..`   
　　▶️Javaで別の部品を使うための読込み。  
　　▶️`@Entity`や`@Id`、`LocalDate`は読み込まないと使えない。  
  
- `@Entity`   
　　▶️このクラスをDBテーブルとして扱う注釈。  
  
- `public class Node`  
　　▶️Nodeというクラスを作っている。  
　　▶️DB設計でいう`Nodeテーブル`に対応する。  
  
- `@ID`  
　　▶️このフィールドの主キーであるという注釈。  
  
- `@GeneratedValue(strategy = GenerationType.IDENTITY)`  
　　▶️idをDB側で自動採番する。  
  
- `private Long id;`  
　　▶️ノード1件を識別する番号  
  
- `private String title;`  
　　▶️単元名。例：二分探索、JOIN、Git branch  
  
- `private String content;`  
　　▶️学習内容の本文。  
  
- `private LocalDate learnedAt;`  
　　▶️学習した日付。  
  
- `private LocalDateTime createdAt;`  
　　▶️データ作成した日時

## Repository
  


### 実装する

```java

package com.example.reviewapp.repository;

import com.example.reviewapp.entity.Node;
import org.springframework.data.jpa.repository.JpaRepository;

public interface NodeRepository extends JpaRepository<Node, Long> {

}

```

▶️解説  
①インターフェースとは？  
→設計図  
classとの違い
```java
// クラス
public class Sample {

}
```
```java
// インターフェース
public interface Sample {

}
```
違いは・・・
```
class → 自分で中身を書く
interface → 中身をSpringが作る
```
②なぜインターフェースを使うのか？  
通常の書き方
```
public List<Node> findAll() {
    // SQL書く
}
```
SpringDateJPAの場合
```
public interface NodeRepository extends JpaRepository<Node, Long> {

}
```
なぜ？
```
Springが裏で実装してくれるから
```
具体的なイメージ
```
① interfaceでNodeRepositoryを書く
↓
② Springが起動する
↓
③ JpaRepositoryを継承して、Node（テーブル名）、Long（主キーの型）を指定する
↓
④ 指示を受けてSpringが「じゃあ実装作っとくね」と裏で実装する
↓
⑤ NodeRepositoryが完成
```
つまり
```
NodeRepositoryは元々存在していない
```

③extendsって何？
```java
extends JpaRepository<Node, Long>
```
意味
```
JpaRepositoryの機能を使います
```
④<Node,Long>の意味
```java
JpaRepository<Node, Long>
```
```
Node → どのテーブル？
Long → 主キーの型
```
まとめ
```java
public interface NodeRepository extends JpaRepository<Node, Long> {

}
```
```
Nodeテーブルを操作するクラスをSpringに作ってもらう
```
