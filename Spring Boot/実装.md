# 実装

- 実際にコードを書く

---

## 最初の目標

`/nodes にアクセスしたら、ノード一覧画面が表示される`

▶️最初はデータが空でもOK

---

## 実装順

最初は以下のような書き方を意識する。  
```
1. Entity：Node.java
2. Repository：NodeRepository.java
3. Service：NodeService.java
4. Controller：NodeController.java
5. View：templates/nodes/list.html
```
理由は、下から積み上げると理解しやすいから。  
```
DBの形  （entity）
↓
DB操作  （repository）
↓
処理  （service）
↓
URL  （controller）
↓
画面  （view）
```
---

## Entity
- 役割：DB設計
  

### 方針

- 実装する前に何が必要かを考える

- クラス：  
Node.javaは、DBの`Node`テーブルに対応するクラス。

- フィールド：
```
id
title
content
learnedAt
createdAt
```
- アノテーション：

```java
@Entity
@Id
@GeneratedValue
```
意味は以下のようなイメージ
```
@Entity → DBテーブルになるクラス
@Id → 主キー
@GeneratedValue → idを自動採番
```


### 実装する

```java

package com.example.reviewapp.entity;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;

import java.time.LocalDate;
import java.time.LocalDateTime;

@Entity
public class Node {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    private String content;

    private LocalDate learnedAt;

    private LocalDateTime createdAt;
}

```


▶️解説  

- `package com.example.reviewapp.entity;`   
　　▶️このファイルが`entity`フォルダの中にあることを表す。  
  
- `import..`   
　　▶️Javaで別の部品を使うための読込み。  
　　▶️`@Entity`や`@Id`、`LocalDate`は読み込まないと使えない。  
  
- `@Entity`   
　　▶️このクラスをDBテーブルとして扱う注釈。  
  
- `public class Node`  
　　▶️Nodeというクラスを作っている。  
　　▶️DB設計でいう`Nodeテーブル`に対応する。  
  
- `@ID`  
　　▶️このフィールドの主キーであるという注釈。  
  
- `@GeneratedValue(strategy = GenerationType.IDENTITY)`  
　　▶️idをDB側で自動採番する。  
  
- `private Long id;`  
　　▶️ノード1件を識別する番号  
  
- `private String title;`  
　　▶️単元名。例：二分探索、JOIN、Git branch  
  
- `private String content;`  
　　▶️学習内容の本文。  
  
- `private LocalDate learnedAt;`  
　　▶️学習した日付。  
  
- `private LocalDateTime createdAt;`  
　　▶️データ作成した日時

## Repository
- 役割：DBの操作


### 実装する

```java

package com.example.reviewapp.repository;

import com.example.reviewapp.entity.Node;
import org.springframework.data.jpa.repository.JpaRepository;

public interface NodeRepository extends JpaRepository<Node, Long> {

}

```

▶️解説  
①インターフェースとは？  
→設計図  
classとの違い
```java
// クラス
public class Sample {

}
```
```java
// インターフェース
public interface Sample {

}
```
違いは・・・
```
class → 自分で中身を書く
interface → 中身をSpringが作る
```
②なぜインターフェースを使うのか？  
通常の書き方
```
public List<Node> findAll() {
    // SQL書く
}
```
SpringDateJPAの場合
```
public interface NodeRepository extends JpaRepository<Node, Long> {

}
```
なぜ？
```
Springが裏で実装してくれるから
```
具体的なイメージ
```
① interfaceでNodeRepositoryを書く
↓
② Springが起動する
↓
③ JpaRepositoryを継承して、Node（テーブル名）、Long（主キーの型）を指定する
↓
④ 指示を受けてSpringが「じゃあ実装作っとくね」と裏で実装する
↓
⑤ NodeRepositoryが完成
```
つまり
```
NodeRepositoryは元々存在していない
```

③extendsって何？
```java
extends JpaRepository<Node, Long>
```
意味
```
JpaRepositoryの機能を使います
```
④<Node,Long>の意味
```java
JpaRepository<Node, Long>
```
```
Node → どのテーブル？
Long → 主キーの型
```
まとめ
```java
public interface NodeRepository extends JpaRepository<Node, Long> {

}
```
```
Nodeテーブルを操作するクラスをSpringに作ってもらう
```

## Service
- Controllerから依頼を受けて、Repositoryを使って処理する場所

### 実装
```java
@Service
public class NodeService{
    private final NodeRepository nodeRepository;

    public NodeService(NodeRepository nodeRepository) {
        this.nodeRepository = nodeRepository;
    }

    public List<Node> findAll() {
        return nodeRepository.findAll();
    }
}
```
▶️解説

```java
@Service
public class NodeService{
    private final NodeRepository nodeRepository;

    public NodeService(NodeRepository nodeRepository) {
        this.nodeRepository = nodeRepository;
    }

    public List<Node> findAll() {
        return nodeRepository.findAll();
    }
}
```

---

```java
@Service
```

このクラスを **ServiceクラスとしてSpringに登録する印**   
「このクラスは処理を書くクラスですよ」とSpringに伝えている

---

```java
public class NodeService{
```

`NodeService` というクラスを作っている
`Node` に関する処理を書く場所

例：

```text
ノード一覧を取得する
ノードを保存する
ノードを削除する
```

---

```java
private final NodeRepository nodeRepository;
```

`NodeRepository` を使うための変数  

`NodeRepository` はDB操作を担当する  

つまりこの行は、  

```text
NodeServiceの中でDB操作係を使えるように準備している
```

という意味  

`private` は、このクラスの中だけで使うという意味  
`final` は、一度入れたら別のものに入れ替えないという意味  

---

```java
public NodeService(NodeRepository nodeRepository) {
```

これは **コンストラクタ**   
`NodeService` が作られるときに、`NodeRepository` を受け取る

イメージはこれ　　

```text
Spring「NodeRepositoryを用意したよ」
NodeService「それを受け取って使います」
```

---

```java
this.nodeRepository = nodeRepository;
```

受け取った `nodeRepository` を、このクラスのフィールドに代入している　　

左側：　　

```java
this.nodeRepository
```

これはクラス内のフィールド　　

右側：　　

```java
nodeRepository
```

これはコンストラクタの引数　　

つまり、　　

```text
外から受け取ったRepositoryを、NodeServiceの中で使えるように保存している
```

という意味  

---

```java
public List<Node> findAll() {
```

`findAll` というメソッドを作っている  
意味は「すべてのNodeを取得する」  

`List<Node>` は、

```text
Nodeが複数入ったリスト
```

という意味  

つまりこのメソッドは、

```text
Node一覧を返すメソッド
```

---

```java
return nodeRepository.findAll();
```

`NodeRepository` の `findAll()` を呼んでいる  

`nodeRepository.findAll()` は、  

```text
DBからNodeを全部取得する
```

処理  

`return` は、その結果を呼び出し元に返す  

つまりこの1行は、  

```text
DBからNode一覧を取得して、そのまま返す
```

という意味  

---

# 全体の流れ

```text
Controller
↓
NodeService.findAll()
↓
NodeRepository.findAll()
↓
DBからNode一覧を取得
↓
Controllerへ返す
```

つまり `NodeService` は、
**ControllerとRepositoryの間に入って、処理を担当するクラス**
