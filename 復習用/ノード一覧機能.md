# 📘 MVC構成 理解まとめ（復習用）

---

# 🧠 問題① Controllerの役割

### ■ 回答（修正後）

```text
Controllerは、ブラウザからのURLを受け取り、
Serviceからデータを取得し、
Modelに詰めてHTMLに渡す役割を持つ
```

---

# 🧠 問題② Serviceの役割

### ■ 回答（修正後）

```text
Serviceは、アプリケーションの処理（ビジネスロジック）を書く場所であり、
Repositoryを使ってデータの取得・加工を行う
```

---

# 🧠 問題③ Repositoryの役割

### ■ 回答

```text
Repositoryは、DBに対する操作（取得・保存・更新・削除）を行う役割
```

---

# 🧠 問題④ ControllerとServiceの違い

### ■ 回答（修正後）

```text
Controllerは、ブラウザからのリクエストを受け取り、
画面表示のためのデータを準備する役割

Serviceは、データの取得や条件処理など、
実際の処理を行う役割
```

---

# 🧠 問題⑤ 処理の流れ

### ■ 回答（修正後）

```text
ブラウザ
↓
Controller
↓
Service
↓
Repository
↓
DB

（結果が戻る）

DB
↓
Repository
↓
Service
↓
Controller
↓
HTML（画面表示）
```

---

# 🧠 問題⑥ なぜServiceを挟むのか

### ■ 回答

```text
・処理を分離するため
・コードの見通しを良くするため
・再利用しやすくするため
・後から修正しやすくするため
```

---

# 🧠 問題⑦ 条件検索はどこに書くか

### ■ 回答（修正後）

```text
Serviceに書く

理由：
条件に応じたデータ取得は「処理」にあたるため、
ControllerではなくServiceに書くべき
```

---

# 🧠 問題⑧ Modelの役割

### ■ 回答

```text
ControllerからHTMLにデータを渡すための箱
```

---

# 🧠 問題⑨ ControllerとHTMLの対応

### ■ 回答

```java
model.addAttribute("nodes", nodes);
```

↓

```html
${nodes}
```

---

```java
return "nodes/list";
```

↓

```text
templates/nodes/list.html
```

---

# 🎯 最終まとめ（超重要）

---

## MVCの役割

```text
Controller：入口＋画面との橋渡し
Service：処理（ロジック）
Repository：DB操作
```

---

## データの流れ

```text
ブラウザ → Controller → Service → Repository → DB
DB → Repository → Service → Controller → HTML
```

---

# 🔥 一番重要な理解

```text
Controllerは「何を表示するか」
Serviceは「どう処理するか」
Repositoryは「どう保存するか」
```
