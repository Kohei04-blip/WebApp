## 必要なデータ一覧

- アプリ上で必要なデータ一覧を考える

### データ一覧を作る時の考え方

  1. 画面をA4用紙に書く
     - 各画面のボタンや操作を紙に書く
    
  2. ユーザー操作を書く
     - １で書いた画面でユーザーが具体的にどんな操作をするか言語化する
  
  3. 保存が必要な情報を抜き出す
     - 操作から保存が必要になるデータを書き出す
  
  4. 一覧にする
     - 一覧にしてまとめる


### 具体例


1. 画面の種類
   1. 復習画面
   2. クイズ画面
   3. ノード画面
   4. ノード作成画面
   5. クイズ作成画面
   6. クイズ一覧画面
   7. 履歴画面


2. ユーザー操作

   1. 復習画面
      - 今日の復習一覧が表示された画面を見る
      - 単元をクリックする
      - 問題画面へ飛ぶ

   2. クイズ画面
      - クイズが表示された画面を見る
      - 回答を入力する
      - 提出する
      - 回答と正解が表示された画面を見る
      - 前の画面に戻る
      - ホームに戻る

   3. ノード画面
      - ノード一覧が表示された画面を見る
      - 単元をクリックする
      - 単元詳細が表示される
      - 編集する
      - 保存する
      - 削除する
      - 前の画面に戻る
      - ホームに戻る

   4. ノード作成画面
      - 画面を見る
      - 分野、単元、内容、理解度、学習日を入力する
      - 入力を保存する
      - 入力を削除する
      - 入力を編集する

   5. クイズ作成画面
      - 空欄に入力する
      - 入力を保存する
      - 入力を削除する
      - 入力を編集する
      - ホームに戻る

   6. クイズ一覧画面
      - クイズが表示された一覧画面を見る
      - 単元をクリックする
      - 前の画面に戻る
    
   7. 復習履歴画面 
      - 問題数、正答数、誤答数の表示数を見る
      - 各項目をクリックする
      - 各項目一覧が表示される
      - ホームに戻る

    

3. 保存が必要なデータ

    - 回答結果
    - 問題文
    - 復習日
    - 正解
    - ノード理解度
    - 復習理解度
    - 分野
    - 単元
    - 内容
    - 学習日
    - 次回復習日


4. 一覧にする

    【ノード系】
    - 分野（subject）
    - 単元（title）
    - 内容（content）
    - 理解度（node_level）
    - 学習日（learned_at）
    
    【クイズ系】
    - 問題文（question）
    - 正解（answer）
    - ノードID（node_id）
    
    【復習系】
    - 回答結果（correct / incorrect）
    - 理解度（review_level）
    - 回答日時（reviewed_level）
    - 次回復習日（next_review_date）

## テーブル設計

### 作り方

  1. データをグループ分けする
     - 大枠のグループにデータを分ける
  
  2. テーブルを作る
     - テーブル名を決める
  
  3. カラム（項目）を決める
     - 各テーブルに必要なカラム（項目）を決める
  
  4. 関係（リレーション）を決める
     - テーブル同士の繋がりを決める
    

【カラムの考え方】  
このテーブル1件を保存するのに必要な情報は何か？

【リレーションの考え方】  
A1件に対してBは何件あるか？
1対多なら、多い側に相手のidを持たせる
  

### 具体例

1. データグループ分け
  - ノード系：Node
  - クイズ系：Quiz
  - 履歴系：ReviewHistory
  - 分野：Subject


2. テーブル&カラム作成
  
  - ノード系：Node
    - 1件のノードを保存するには何が必要か？
      - 識別：id
      - 単元名：title
      - 内容：content
      - どの分野か：subject_id
      - 作成日：created_id
      - 学んだ日：learned_at
      - 理解度：understanding_level
  
  - クイズ系：Quiz
    - 1件のノードを保持するには何が必要か？
      - 識別：id
      - どのノードに属するか：node_id
      - 問題：question
      - 正解：answer
      - 次回の復習日：next_review_date
      - 作成日：created_at
      - 更新日：updated_id
  
  - 履歴系：ReviewHistory
    - 1件の履歴を保存するには何が必要か？
      - 識別：id
      - どのクイズか：quiz_id
      - 結果：result
      - 理解度：understanding_level
      - 復習日：reviewed_at
  
  - 分野：Subject
    - 1件の分野を保持するには何が必要？
      - 識別：id
      - 分野名：name（java,sqlなど）


3. リレーション決定
  - Subject 1 : N Node
  - Node 1 : N Quiz
  - Quiz 1 : N ReviewHistory

## クラス設計（Entity設計）

### ■ クラス設計とは
DBで設計したテーブルを、Javaのオブジェクトとして表現すること

---

### ■ 基本ルール

#### ルール①：テーブル = クラス
- Nodeテーブル → Nodeクラス
- Quizテーブル → Quizクラス
- ReviewHistoryテーブル → ReviewHistoryクラス
- Subjectテーブル → Subjectクラス

---

#### ルール②：カラム = フィールド
- title → String title
- content → String content
- learned_at → LocalDate learnedAt

---

#### ルール③：リレーション = オブジェクト参照

■ DB
Quizテーブル
- node_id

■ Java
class Quiz {
    Node node;
}

→ IDではなく「オブジェクト」で持つ

---

### ■ クラス設計（例）

#### Subjectクラス

class Subject {
    Long id;
    String name;
}

#### Nodeクラス

class Node {
    Long id;
    String title;
    String content;
    Subject subject;
    LocalDate learnedAt;
}

#### Quizクラス

class Quiz {
    Long id;
    Node node;
    String question;
    String answer;
    LocalDate nextReviewDate;
}

#### ReviewHistoryクラス

class ReviewHistory {
    Long id;
    Quiz quiz;
    Boolean result;
    Integer understandingLevel;
    LocalDateTime reviewedAt;
}


## 処理一覧の考え方

### ■ 処理一覧とは
画面でユーザーが行う操作を、システムが実行する処理に変換した一覧のこと

---

### ■ 目的
- 実装前に必要な機能を整理する
- どのクラスが何を担当するか明確にする
- 画面設計からService設計につなげる

---

### ■ 基本の考え方

### ① 画面の目的を考える
- この画面は何のためにあるのかを明確にする

例：
- ノード画面 → ノード一覧や詳細を確認する
- クイズ画面 → クイズに回答する
- 復習履歴画面 → 過去の結果を見る

---

#### ② ユーザーの操作を考える
- ユーザーは画面上で何をするのかを書く

例：
- 一覧を見る
- ボタンを押す
- 入力する
- 保存する
- 削除する

---

#### ③ システムの処理に変換する
- ユーザー操作に対して、システム側で必要な処理を書く

例：
- 一覧を見る → 一覧データを取得する
- 保存する → 入力値を受け取って保存する
- 回答する → 正誤判定して履歴を保存する

---

### ■ 基本の流れ

画面
↓
ユーザー操作
↓
必要な処理


---


### 具体例

#### 分野
- 分野一覧を取得する
- 分野に紐づくノード一覧を取得する

---

#### ノード画面
- ノード一覧を取得する
- ノード詳細を取得する
- 入力値を受け取り、ノードを保存する
- ノードを更新する
- ノードを削除する

---

#### クイズ画面
- クイズを取得する
- 入力値（回答）を受け取る
- 回答内容と正解を比較して正誤判定する
- 正解とユーザー回答を表示する
- 理解度の入力値を受け取り、保存する
- 回答結果を履歴として保存する
- 理解度に応じて次回復習日を更新する
- 次のクイズを取得する

---

#### 履歴画面
- 復習履歴一覧を取得する
- 正答数を集計する
- 誤答数を集計する
- 履歴詳細を取得する
