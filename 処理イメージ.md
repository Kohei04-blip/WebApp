# 全体の処理フロー

## イメージ図
```mermaid
flowchart TD
    A[ユーザー操作（ブラウザ）]
    B[Controller（入口）]
    C[Service（処理・ロジック）]
    D[Repository（DB操作）]
    E[Database（DB）]

    A --> B
    B --> C
    C --> D
    D --> E
    E --> D
    D --> C
    C --> B
    B --> A
```

## イメージ
```
① ユーザーが操作する
② API（Controller）が受け取る
③ Serviceが処理する
④ DBに保存/取得する
⑤ 結果を画面に返す
```


## より直感的なイメージ
```
ユーザー
  ↓
「ノード一覧見たい！」

Controller（受付）
  ↓
「一覧くれって来てるよ」

Service（頭脳）
  ↓
「じゃあDBから取ってこよう」

Repository（窓口）
  ↓
「DBからデータ持ってきたよ」

DB（倉庫）

→ 戻る

Service
  ↓
Controller
  ↓
画面
  ↓
ユーザーに表示
```
